# SpeedDic 产品需求文档 (PRD)

## 1. 产品概述
SpeedDic 是一款融合了 AI 技术与碎片化学习理念的桌面端语言学习工具。它通过将待学习单词无缝集成到 Windows 桌面壁纸中，利用用户在切换窗口或回到桌面的间隙进行“无感”记忆，结合经典的间隔重复算法（SRS），实现高效的词汇积累。

## 2. 核心功能需求

### 2.1 AI 场景化单词生成
*   **功能描述**：利用大模型 API（如 Gemini, OpenAI 等）根据用户设定的“场景”（如：互联网架构、商务谈判、西餐礼仪）生成相关的英文单词。
*   **数据项**：单词、音标、中文含义、例句（可选）、场景分类。
*   **交互**：用户可输入自定义场景关键词，系统调用 API 生成一批单词（如 10-20 个）并存入本地库

### 2.2 桌面壁纸嵌入 (Active Wallpaper)
*   **功能描述**：将当前待学习的单词卡片以透明或半透明层的方式嵌入到 Windows 桌面壁纸中。
*   **技术要求**：
    - 不影响桌面图标的操作。
    - 支持壁纸自适应，单词卡片位置、大小、透明度可配置。
    - 实时刷新：当用户标记单词后，桌面内容立即更新。

### 2.3 学习交互操作
*   **操作选项**：
    1.  **已掌握 (Mastered)**：单词进入长期记忆库，减少复现频率。
    2.  **模糊 (Vague)**：单词在短时间内再次复现（如 1 天后）。
    3.  **不清楚 (Unclear)**：单词在极短时间内多次复现（如 10 分钟后或下一次刷新）。
*   **交互方式**：通过桌面悬浮小部件、系统托盘菜单或全局快捷键进行标记。

### 2.4 本地数据库与 SRS 复习系统
*   **数据存储**：使用 SQLite 或类似本地数据库存储单词、记忆状态、最后复习时间、复习次数等。
*   **复习算法**：参考 Anki (SM-2 算法) 或 莱特纳盒 (Leitner System)，根据用户的掌握程度动态计算下一次“待复习”的时间点。
*   **每日任务**：系统每天自动根据算法输出“今日待复习”列表，并优先显示在桌面壁纸上。
*   **模型可读**：整个的复习系统存储在merory.md中，可供模型读取、修改

### 2.5 查看历史学习记录
*   **交互方式**：通过桌面悬浮小部件，点击“查看学习记录”，展示所有已背单词与下一次复习时间的页面
*   **操作反馈**：列表支持滑动查看，并提供搜索/筛选功能，方便查找特定单词。
*   **页面内容展示**
    - **A. 数据概览（Dashboard）**
      1. **累计背诵**：展示已完成学习的单词总数。
      2. **今日待复习**：统计当前时刻已到达复习节点的单词量。
      3. **掌握程度**：通过环形图展示（如：陌生、模糊、熟练、牢记）。
    - **B. 详细列表（Word Log）**
        **采用时间轴或列表形式展示具体词汇**：列标题有：单词/词组、释义、音标、学习状态、复习倒计时
        | 字段 | 说明 | 示例 |
        | :--- | :--- | :--- |
        | 单词/词组 | 已学习的原文及简要释义 | Persistence (持续/毅力) |
        | 学习状态 | 标识当前记忆阶段 | 第 3 次复习完成 |
        | 复习倒计时 | 增强紧迫感（可选） | 还有 18 小时 |

## 3. UI/UX 设计要点
*   **极简主义**：单词卡片不应喧宾夺主，干扰用户正常工作。
*   **视觉反馈**：点击掌握/模糊时，应有轻微的动画反馈。
*   **场景化 UI**：不同场景可搭配不同的壁纸主题或配色方案。

## 4. 路线图 (Roadmap)
- **V1.0**：实现本地数据库管理、简单的 AI 单词生成及桌面基本显示。
- **V2.0**：完善 SRS 算法，支持自定义壁纸背景及字体样式。
