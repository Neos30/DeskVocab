# SpeedDic 产品需求文档 (PRD)

## 1. 产品概述
SpeedDic 是一款融合了 AI 技术与碎片化学习理念的桌面端语言学习工具。它通过将待学习单词无缝集成到 Windows 桌面壁纸中，利用用户在切换窗口或回到桌面的间隙进行“无感”记忆，结合经典的间隔重复算法（SRS），实现高效的词汇积累。

## 2. 核心功能需求

### 2.1 AI 场景化单词生成
*   **功能描述**:
    1. 利用大模型 API（如 Gemini, OpenAI 等）根据用户设定的“场景”（如：互联网架构、商务谈判、西餐礼仪）生成相关的英文单词。
    2. 利用大模型 API（如 Gemini, OpenAI 等）根据用户导入的文章（pdf、docx、doc、txt），统计出现频率较高的单词并排除常用主语、介词等连接词
*   **数据项**：单词、音标、中文含义、例句（可选）、场景分类。
*   **交互**:
    1. 用户可输入自定义场景关键词，系统调用 API 生成一批单词（如 10-20 个）并存入本地库
    2. 用户导入一篇文档（支持 pdf、docx、doc、txt 格式），填入统计要求，调用 API 生成一批单词（如 10-20 个）并存入本地库
*   **操作流程与状态约束**:
    - 设置页面将"模型配置保存"、"选择场景生成新词"、"导入文档生成新词"三项操作在视觉与逻辑上进行隔离。
    - **前置条件**：用户必须先完成模型配置（填写 API Key、选择模型、点击"保存配置"并验证通过）后，"选择场景生成新词"与"导入文档生成新词"按钮才变为可点击状态；在配置未保存或验证失败时，两个生成按钮保持禁用（灰化），并提示"请先保存模型配置"。

### 2.2 桌面壁纸嵌入 (Active Wallpaper)
*   **功能描述**：将当前待学习的单词卡片以透明或半透明层的方式嵌入到 Windows 桌面壁纸中。
*   **技术要求**：
    - 不影响桌面图标的操作。
    - 支持壁纸自适应，单词卡片位置、大小、透明度可配置。
    - 实时刷新：当用户标记单词后，桌面内容立即更新。

### 2.3 学习交互操作
*   **操作选项**：
    1.  **已掌握 (Mastered)**：进入 SRS 长期记忆队列，算法降低复现频率（对应 Dashboard 中"熟练"状态）。
    2.  **模糊 (Vague)**：进入 SRS 短期复习队列，约 1 天后再次复现（对应"模糊"状态）。
    3.  **不清楚 (Unclear)**：进入 SRS 高频复习队列，约 10 分钟后或下一次刷新时复现（对应"不清楚"状态）。
    4.  **跳过**：直接标记为"掌握"状态，**仅记录、不进入 SRS**，后续不再安排复习（对应 Dashboard 中"牢记"状态）。
*   **交互方式**：通过桌面悬浮小部件、系统托盘菜单或全局快捷键进行标记。

### 2.4 本地数据库与 SRS 复习系统
*   **数据存储**：使用 SQLite 或类似本地数据库存储单词、记忆状态、最后复习时间、复习次数等。
*   **复习算法**：参考 Anki (SM-2 算法) 或 莱特纳盒 (Leitner System)，根据用户的掌握程度动态计算下一次”待复习”的时间点。三种操作对应的算法行为如下：
    - **已掌握** → 算法拉长复习间隔（长期低频）
    - **模糊** → 算法设置约 1 天后复习
    - **不清楚** → 算法设置约 10 分钟后复习
    - **跳过** → 不经过 SRS，直接置为”掌握”，不再参与复习调度
*   **每日任务**：系统每天自动根据算法输出“今日待复习”列表，并优先显示在桌面壁纸上。
*   **模型可读**：整个的复习系统存储在merory.md中，可供模型读取、修改

### 2.5 查看历史学习记录
*   **交互方式**：通过桌面悬浮小部件，点击“查看学习记录”，展示所有已背单词与下一次复习时间的页面
*   **操作选项**：
    1. 列表支持滑动查看，并提供搜索/筛选功能，方便查找特定单词
    2. 当学习记录单词为`待复习`状态时，点击同步按钮，会将待复习的单词同步至桌面展示栏中
*   **页面内容展示**
    - **A. 数据概览（Dashboard）**
      1. **累计背诵**：展示已完成学习的单词总数。
      2. **今日待复习**：统计当前时刻已到达复习节点的单词量。
      3. **掌握程度**：通过环形图展示（如：不清楚、模糊、熟练、牢记）。各状态与 2.3 操作的对应关系如下：
         1. **牢记**：通过"跳过"操作标记的单词，不再参与复习
         2. **熟练**：通过"已掌握"操作进入 SRS 的单词，复习频率低
         3. **模糊**：通过"模糊"操作进入 SRS 的单词，复习频率中
         4. **不清楚**：通过"不清楚"操作进入 SRS 的单词，复习频率最高
    - **B. 详细列表（Word Log）**
        **采用时间轴或列表形式展示具体词汇**：列标题有：单词/词组、释义、音标、学习状态、复习倒计时
        | 字段 | 说明 | 示例 |
        | :--- | :--- | :--- |
        | 单词/词组 | 已学习的原文及简要释义 | Persistence (持续/毅力) |
        | 学习状态 | 标识当前记忆阶段 | 第 3 次复习完成 |
        | 复习倒计时 | 增强紧迫感（可选） | 还有 18 小时 |

## 3. UI/UX 设计要点
*   **极简主义**：单词卡片不应喧宾夺主，干扰用户正常工作。
*   **视觉反馈**：点击掌握/模糊时，应有轻微的动画反馈。
*   **场景化 UI**：不同场景可搭配不同的壁纸主题或配色方案。

## 4. 路线图 (Roadmap)
- **V1.0**：实现本地数据库管理、简单的 AI 单词生成及桌面基本显示。
- **V2.0**：完善 SRS 算法，支持自定义壁纸背景及字体样式。
